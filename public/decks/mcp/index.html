Problem framing

Why a protocol?

Without a common protocol, every AI client and every tool provider needs custom integration work. A shared protocol creates a more reusable integration surface.

BeforePoint-to-point integrations and bespoke plugins.
AfterClients can discover server capabilities consistently.
Teaching payoffIt gives learners a concrete bridge from agents to enterprise systems.

Key model

Resources, tools, and prompts

This is the mental model most learners need first: resources expose context, tools expose actions, and prompts expose reusable interaction patterns.

ResourcesReadable context exposed by a server.
ToolsCallable operations exposed by a server.
PromptsReusable templates or workflows exposed by a server.

Vertical dive

Example: support knowledge system

A support MCP server might expose a product manual as a resource, a ticket lookup as a tool, and a reusable “draft customer reply” prompt.

MVP implementation note

How this prototype links decks

Cross-deck links pass returnTo and returnLabel in the query string. Each deck reads those values and shows a return button when available.

Directly open the originating Agents slide This link is hardcoded; the nav bar return link is dynamic.
