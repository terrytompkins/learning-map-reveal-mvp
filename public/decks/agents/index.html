AI Agents

An agentic system uses a model to decide what to do next, call tools, inspect results, and continue toward a goal.

The loop does not need to be fully autonomous. For many enterprise workflows, the best agent is constrained, observable, and human-steered.

Important mechanism

Tool calling

Tool calling lets the model request a software operation: search, fetch a record, run a calculation, update a ticket, call an API, or generate a file.

ModelChooses a tool and fills arguments.
RuntimeValidates and executes the tool call.
ModelUses the result to continue or answer.

Vertical dive

Tool calls need guardrails

Treat model-proposed actions as requests, not as trusted commands. Validate inputs, scope permissions, log calls, and require human approval for high-impact actions.

System behavior

Memory is not one thing

Agent memory can mean conversation state, working notes, user preferences, task history, retrieved documents, or persistent domain knowledge.

Short-termWhat is happening in the current interaction?
Long-termWhat should the system remember across sessions?
ExternalWhat should be retrieved from source systems?

Quality loop

Evaluation & governance

Agentic behavior is harder to evaluate than a single prompt response because the system can take multiple paths. Use task-level success criteria, trace inspection, tool-call review, and regression tests.

Task successDid it complete the user goal?
Process qualityWere the steps safe and explainable?
RegressionDoes it keep working as prompts, tools, and models change?
